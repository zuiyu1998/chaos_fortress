# Chaos Fortress — 游戏设计文档

## 1. 概述 (Overview)

- **游戏名称**: Chaos Fortress
- **游戏类型**: 2D 塔防/肉鸽/自走棋
- **目标平台**: 桌面端（Windows / macOS / Linux），未来可扩展至 Web (WASM)
- **开发引擎**: Bevy 0.18（Rust，纯 2D 模式）
- **目标受众**: 喜爱 Roguelite / 塔防 / 自走棋的玩家
- **核心卖点**: 丰富的构筑系统/割草

### 1.1 一句话描述

玩家扮演一个军团长，通过敌人掉落的资源在商店强化自己的实力，使用招募的角色守住要塞，直到调令下达。

---

## 2. 核心玩法 (Core Gameplay)

- **核心循环**
    - 玩家进入 > 战斗/收集 > 商店强化 > 挑战升级/退出 > 循环
- **操作方式**
    - 键盘 + 鼠标（移动、攻击、交互、菜单导航）
- **胜利与失败条件**
    - 胜利条件: 击败所有敌人
    - 失败条件: 要塞被攻破
- **难度曲线**
    - 根据当前基地的总价值决定

## 9. 技术架构 (Technical Architecture)

- **引擎与语言**
    - Bevy 0.18（Rust 2024 edition），2D 模式
- **项目模块结构**
    - `src/main.rs` — 入口、App 组装
    - `src/screens/` — 游戏场景（Splash、Title、Loading、Gameplay）
    - `src/menus/` — 菜单界面（Main、Pause、Settings、Credits）
    - `src/audio.rs` — 音频系统
    - `src/theme/` — UI 主题样式
    - `src/dev_tools.rs` — 开发调试工具
    - `src/asset_tracking.rs` — 资源追踪
    - `src/demo/` — 演示/测试场景
- **关键设计决策**
    - 为何选择 Bevy（ECS 架构、Rust 性能、开源生态）
